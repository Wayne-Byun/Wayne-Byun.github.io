세부 설정 변경, 포스트 작성, 깃헙페이지에 배포

만들어진 디렉토리를 살펴보면 여러 파일과 하위 디렉토리가 보인다.

// 디렉토리 구조
blog/
  build/
  public/
  src/
    build/
    component/
    constants/
    hooks/
    images/   <------------- favicon 이미지
    pages/
    posts/    <------------- 포스트 작성
    store/
    style/
    templates/
    utils/
  gatsby-config.js      <--- 메뉴 이름
  gatsby-meta-config.js <--- 블로그 이름, 링크
  ...
  README.md

스타터가 제공해주는 디자인이나 기능은 지금으로도 충분하므로 몇가지 설정을 바꾸고 src/posts에 포스트와 그에 들어가는 이미지를 넣어주면 된다. (하나씩 뜯어보면서 확인해도 되겠지만 이미 README.md 파일에 잘 작성되어있다.)

// gatsby-meta-config.js
module.exports = {
  title: '"일해라 변과장"의 블로그', // Your website title
  description: `Wayne's Blog`, // Your website description
  author: 'Wayne-Byun', // Maybe your name
  siteUrl: 'https://wayne-byun.github.io', // Your website URL
  lang: 'ko', // Language
  utterances: 'Wayne-Byun/blog-comment', // Github repository to store comments
  links: {
    github: 'https://github.com/Wayne-Byun', // Your github repository
  },
  icon: 'src/images/icon.png', //  Favicon Path
};

블로그 서비스는 깃헙 페이지를 이용할 것이므로 해당 url로 작성하였고 utterances는 나중에 댓글 기능을 추가할 때 다시 살펴보면 될 것 같다.

이제 posts 디렉토리에 글만 작성하면 된다.

### "2021년 7월 8일"

인사부로 전입한 후 어느새 1년이 흘렀다.  
은행에 입행한 후 IT가 아닌 부서는 처음이라 걱정을 많이 했지만,  
새로운 사람들과 새로운 일을 해온 지난 시간을 돌아보니 걱정보단 즐거운 나날이었다.

샘플로 작성되어있는 md파일을 참고하여 정보를 넣고 markdown으로 글을 작성하면 된다.

gatsby develop커맨드가 실행되어있으면 변경사항을 반영하여 자동으로 페이지를 생성해준다.

카테고리가 1개만 지정할 수 있도록 되어있는데 나중에 필요하면 이부분은 여러개를 지정 할 수 있도록 바꿔봐도 좋을것 같다.

이제 만들어진 페이지를 깃헙 페이지에 올리기만 하면 된다.
기본적으로 git이 설치되어 있어야 하고 깃헙 페이지 repository에 연결이 되어있어야 한다.

gh-page를 설치하고 package.json에 deploy 스크립트를 추가한다.

npm install -d gh-page
// packege.json
...
  "scripts": {
    "build": "gatsby build",
    "develop": "gatsby develop",
    "format": "prettier --write \"**/*.{js,jsx,ts,tsx,json,md}\"",
    "start": "npm run develop",
    "serve": "gatsby serve",
    "clean": "gatsby clean",
    "deploy": "gatsby build && gh-pages -d public -b master" // deploy 스크립트
  }
...

이제 deploy 스크립트로 깃헙에 배포하면 끝.

npm run deploy
